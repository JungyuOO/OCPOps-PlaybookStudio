Official Corpus Text Layer Correction

작성일: 2026-05-17 / 대상: v0.1.4 official corpus, Qdrant payload, embedding projection

한 줄 결론: 이전 작업에서는 표시용 text와 벡터용 embedding_text가 섞였고, base64 출력 예시가 임베딩 입력에 들어갔다. 현재는 text=표시용, normalized_text=BM25용, embedding_text=벡터용으로 분리했고, J가 지적한 Tk9ER... 노이즈는 검색/임베딩 계층에서 제거했다.

무엇이 잘못됐나

문제 1

표시용 text가 embedding_text처럼 저장됨

embedding_chunks.jsonl.text와 Qdrant payload.text가 벡터 입력용 flat text로 들어갔다.

결과적으로 답변 화면에서 개행, 코드블록, 표 구조가 사라질 수 있었다.

문제 2

base64 출력 예시가 임베딩에 포함됨

Tk9ERUlQX0hJTlQ9MTkyLjAuMCxxxx== 같은 코드블록 출력값이 embedding_text에 들어갔다.

추가 재검토에서는 padding이 없는 Tk9ERUlQX0hJTlQ9MTkyLjAuMC 형태도 같은 계열의 노이즈로 확인했다.

원문 표시에는 필요할 수 있지만 의미 검색에는 노이즈다.

오해 가능

청크 사이즈를 새로 준 작업은 아님

기존 chunks.jsonl의 chunk_id 기준 projection 작업이었다.

row 수가 줄어든 이유는 empty/duplicate/contained overlap 제외 때문이다.

정상 계약

필드 정상 역할 예상 형태
raw_text 원본 보존 지저분해도 원문 그대로 유지
text / markdown 사람과 챗봇 화면 표시용 개행, 코드블록, 표, 백틱 유지 가능
normalized_text BM25/keyword 검색용 기호와 마크업을 많이 제거한 flat text
embedding_text vector embedding 입력용 flat text. 마크업, URL, base64 출력 예시는 제거하되 명령어 의미 기호는 유지

무엇을 고쳤나

파일 수정 내용
src/play_book_studio/ingestion/official_gold_import.py embedding_chunks.jsonl.text를 표시용 markdown으로 저장하도록 수정. embedding_text와 normalized_text에서는 base64-like token 제거.
src/play_book_studio/db/qdrant_indexer.py Qdrant payload.text는 markdown 우선으로 채우고, text_fields.embedding_text는 벡터용 필드로 분리.
src/play_book_studio/ingestion/official_embedding_qdrant.py 더 이상 payload.text == text_fields.embedding_text를 요구하지 않도록 품질 게이트 수정. 대신 embedding field mismatch, empty display text, raw_text leak 등을 검사.
tests/test_official_gold_import.py, tests/test_qdrant_indexer.py 표시용 text와 embedding_text 분리, base64 출력 제거, Qdrant payload 계약을 테스트로 고정.

7. 전체 Qdrant official docs scan

@'
import json, re, urllib.request

body = {
    "limit": 2000,
    "with_payload": True,
    "with_vector": False,
    "filter": {
        "must": [
            {"key": "source.corpus_scope", "match": {"value": "official_docs"}}
        ]
    },
}

url = "http://127.0.0.1:6335/collections/openshift_docs/points/scroll"
base64_re = re.compile(r"(?<![A-Za-z0-9+/])(?:[A-Za-z0-9+/]{20,}={0,2})(?![A-Za-z0-9+/])")

def is_likely_base64_token(value):
    token = str(value or "").rstrip("=")
    return (
        len(token) >= 20
        and any(char.isupper() for char in token)
        and any(char.islower() for char in token)
        and any(char.isdigit() for char in token)
    )

counts = {
    "checked": 0,
    "embedding_newline": 0,
    "embedding_marker": 0,
    "embedding_base64_like": 0,
    "normalized_newline": 0,
    "raw_text_leak": 0,
    "empty_payload_text": 0,
    "payload_embedding_mismatch": 0,
}

offset = None
while True:
    if offset is not None:
        body["offset"] = offset
    req = urllib.request.Request(
        url,
        data=json.dumps(body).encode("utf-8"),
        headers={"Content-Type": "application/json"},
        method="POST",
    )
    data = json.loads(urllib.request.urlopen(req, timeout=60).read().decode("utf-8"))["result"]
    for point in data.get("points", []):
        payload = point.get("payload") or {}
        fields = payload.get("text_fields") or {}
        embedding_text = str(fields.get("embedding_text") or "")
        normalized_text = str(fields.get("normalized_text") or "")
        counts["checked"] += 1
        if not str(payload.get("text") or "").strip():
            counts["empty_payload_text"] += 1
        if "\n" in embedding_text or "\t" in embedding_text or "\r" in embedding_text:
            counts["embedding_newline"] += 1
        if any(marker in embedding_text for marker in ("[CODE", "[/CODE]", "[TABLE", "[/TABLE]", "```")):
            counts["embedding_marker"] += 1
        if any(is_likely_base64_token(match.group(0)) for match in base64_re.finditer(embedding_text)):
            counts["embedding_base64_like"] += 1
        if "\n" in normalized_text or "\t" in normalized_text or "\r" in normalized_text:
            counts["normalized_newline"] += 1
        if "raw_text" in json.dumps(payload, ensure_ascii=False):
            counts["raw_text_leak"] += 1
    offset = data.get("next_page_offset")
    if not offset:
        break

print(json.dumps(counts, ensure_ascii=False, indent=2))

참고

추가 재검토에서 padding 없는 base64-like token 4건을 더 발견했고, 필터와 Qdrant 품질 게이트를 확장한 뒤 DB/Qdrant에서 0건으로 재확인했다.

아래 항목은 이번 수정의 실패가 아니라, 다음 작업자가 과하게 완료로 이해하지 않도록 남기는 후속 확인 대상이다.

  • 반복이 많은 표 chunk는 normalized_text == embedding_text가 될 수 있다. 이는 명령어/코드가 없는 표 청크에서는 오류가 아닐 수 있다.
  • 다만 반복 표는 rank에서 낮게 보거나 metadata로 table 성격을 반영하는 후속 개선 후보가 될 수 있다.
  • BM25 runtime loader는 후속 확인/수정이 필요하다. 현재 구조상 normalized_text를 항상 직접 토큰화한다고 단정하면 안 된다.
  • docker compose seed 경로가 기존 collection에 대해 payload만 refresh하고 vector를 재임베딩하지 않는 경로를 탈 수 있는지 추가 확인이 필요하다.
  • 새 chunk size를 적용한 rechunk 작업은 아니다. chunk size 정책을 바꾸려면 별도 협의가 필요하다.
  • text에는 코드블록과 개행이 남는 것이 정상이다. 임베딩 품질 검사는 text_fields.embedding_text 기준으로 본다.
