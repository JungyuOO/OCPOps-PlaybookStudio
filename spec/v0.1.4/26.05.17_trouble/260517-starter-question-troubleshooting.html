첫 화면 추천 질문 문구 개선 트러블슈팅

작성일: 2026-05-17 / Branch: feat/corpus_embedding_test_S_v.2 / Head: c175055

문제 첫 화면 추천 질문의 한국어가 어색해 실제 운영자가 누르고 싶은 질문처럼 보이지 않았다.
원인 answer LLM 문제가 아니라 starter question API의 후보 질문/템플릿 문구 문제였다.
수정 표시 직전 보정과 청크 후보 생성 템플릿을 함께 수정했다.
검증 관련 테스트 18건 통과, seed 4개 스모크에서 금지 표현 0건 확인.

1. 접수된 화면과 피드백

아래 화면은 페이지 첫 화면에 노출되는 시작 질문 카드다. 피드백은 “메인 질문들의 한글이 어색하다”는 내용이었다.

기존 문구 피드백 기준 개선 방향
Service와 Route 연결은 뭔지부터 알고 싶은데 어디서 확인하면 돼? Service와 Route 연결 구조를 먼저 이해하고 싶은데, 어디를 보면 될까요?

2. 원인 판단

첫 화면 질문은 프론트에서 문장을 직접 하드코딩하거나 answer LLM이 즉석 생성하는 구조가 아니었다.

  • 프론트는 apps/web/src/pages/WorkspacePage.tsx에서 API가 내려준 item.question을 그대로 렌더링한다.
  • API 라우트는 /api/studio/starter-questions이고, server_handler_factory.py에서 starter_questions.py로 연결된다.
  • starter_questions.py는 DB의 document_chunks.starter_question_candidates, official manifest, learning manifest를 읽어 질문을 구성한다.
  • 청크 후보 질문은 chunk_question_candidates.py에서 생성되므로, 재적재/재생성 시 어색한 후보가 다시 들어갈 수 있었다.

따라서 이번 수정은 프론트 UI 수정이 아니라 “후보 질문 생성 + 표시 직전 문구 보정”을 함께 잡는 방향으로 진행했다.

3. 수정 정책

질문 유형 적용 문구 정책
개념 학습형 Service와 Route 연결 구조를 먼저 이해하고 싶은데, 어디를 보면 될까요?
상태 확인형 노드 상태를 먼저 확인하려면 어디를 보면 될까요?
명령어 확인형 Node 확인에 먼저 사용할 명령어는 무엇인가요?
문제 해결형 Service 문제가 생겼을 때 원인을 어디서부터 좁히면 좋을까요?
작성/구성형 FRRConfiguration CRD 구성의 기본 구조는 어떻게 잡으면 될까요?

금지 표현은 뭔지부터, 어디서 확인하면 돼, 처음에는 무엇부터, 어떤 명령어부터 쓰면 돼로 두고, starter question 샘플에서는 0건이 되도록 확인했다.

4. 변경 내용

  • starter_questions.py: _starter_question에서 표시 직전 문구 보정 함수를 적용했다. 기존 DB 후보 질문이 어색해도 화면에 나가기 전에 자연스럽게 바뀐다.
  • starter_questions.py: _compose_beginner_question 템플릿을 정중하고 명확한 질문형으로 수정했다.
  • chunk_question_candidates.py: 청크에서 생성되는 starter/follow-up 후보 질문 템플릿도 같은 말투 기준으로 수정했다.
  • 테스트: 기존 말투를 고정하던 기대값을 새 정책 기준으로 바꾸고, 금지 표현이 들어오지 않는지 확인했다.

7. 남은 확인사항

  • 현재 수정은 문구 생성/보정 로직 기준으로 검증했다. 실제 브라우저 첫 화면은 dev server를 띄운 뒤 같은 seed/session에서 한 번 더 육안 확인하면 좋다.
  • DB에 저장된 기존 후보 질문 원문은 이번 작업에서 직접 수정하지 않았다. 대신 표시 직전 보정으로 화면 노출 문구를 개선했다.
  • 향후 corpus를 재적재하면 chunk_question_candidates.py의 개선된 후보 템플릿이 반영된다.
  • RAG 검색, Qdrant 적재, answer LLM, reranker 품질은 이번 문구 개선 범위가 아니므로 별도 테스트 결과와 섞지 않는다.
