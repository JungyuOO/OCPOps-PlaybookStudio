# S worklog

이 폴더는 S가 맡은 데이터, 코퍼스, Wiki Library, 업로드 파이프라인 작업을 잊지 않기 위한 작업 로그입니다.

## 폴더 구조

- `done/`: 실제로 진행했거나 확인한 작업 기록입니다.
- `todo/`: 아직 제품 기준으로 끝나지 않은 후속 작업입니다.

## 기록 원칙

- `done`은 완료 자랑이 아니라, 무엇을 했고 어떤 증거가 있는지 남기는 공간입니다.
- `todo`는 막연한 메모가 아니라, pass/fail 가능한 완료 조건과 확인 방법을 같이 적습니다.
- 숫자, 상태, Gold, 문서 수, 후보 수처럼 혼동이 쉬운 것은 반드시 어떤 API/DB/파일 기준인지 적습니다.
- 팀 협업 이슈는 S/J 역할 경계와 handoff 계약을 같이 남깁니다.
